# 小说改编 AI 漫剧制作清单交互页设计文档

## 1. 目标

创建一个单文件离线 HTML 页面，用于管理“小说改编 AI 漫剧制作清单”。

页面需要满足以下目标：

- 双击即可打开，无需安装环境或启动服务。
- 支持填写和补充项目字段，而不是只做静态展示。
- 支持动态新增和删除重复类内容，如章节、角色、场景、道具、分镜、台词。
- 支持本地图片上传与预览，适用于角色图、场景图、道具图等视觉信息。
- 支持浏览器本地自动保存，避免用户刷新或关闭后内容丢失。
- 支持导出 JSON 与导入 JSON，方便继续编辑或迁移到其他设备。
- 兼顾桌面端编辑效率与移动端基础可用性。

## 2. 非目标

本次实现不包含以下能力：

- 不接入后端服务，不做云端同步。
- 不接入数据库或账号系统。
- 不生成 Word、Excel、Notion 原生文件。
- 不自动调用 AI 生成 prompt、分镜或文案。
- 不处理大型图片压缩、裁剪或复杂素材管理。

## 3. 页面形态

采用“标签页工作台”结构，而不是单一长表单。

整体布局如下：

1. 顶部工具栏
2. 概览区
3. 标签导航
4. 当前模块编辑区

### 3.1 顶部工具栏

顶部提供以下操作：

- 新建空白项目
- 恢复默认模板
- 导出 JSON
- 导入 JSON
- 打印或导出 PDF
- 自动保存状态提示

### 3.2 概览区

概览区展示全局关键信息：

- 项目名称
- 小说名称
- 改编章节范围
- 视频比例
- 目标平台
- 进度百分比
- 已完成项数量
- 未完成项数量

### 3.3 标签导航

页面按模块切换，标签包括：

- 项目基础信息
- 制作流程
- 章节改编
- 角色设定
- 场景设定
- 道具特效
- 分镜脚本
- Prompt 模板
- 配音台词
- 音乐音效
- 剪辑执行
- 质量检查
- 文件夹模板
- Demo 开工版

### 3.4 内容组织

每个标签页使用卡片式内容区域。

- 固定信息模块使用表单卡片。
- 可重复内容模块使用“条目列表 + 条目编辑卡片”。
- 清单模块使用复选框分组显示。
- 图片模块在对应条目内显示上传、预览和删除区域。

## 4. 数据模型

页面采用单一 JSON 状态树进行管理，便于保存与导入导出。

顶层结构建议如下：

```json
{
  "projectInfo": {},
  "workflow": {},
  "episodes": [],
  "characters": [],
  "scenes": [],
  "props": [],
  "storyboards": [],
  "prompts": {},
  "voiceLines": [],
  "audio": {},
  "editing": {},
  "quality": {},
  "folders": [],
  "demoChecklist": [],
  "meta": {}
}
```

### 4.1 projectInfo

保存基础信息字段：

- 项目名称
- 小说名称
- 改编章节范围
- 集数规划
- 单集时长
- 视频比例
- 目标平台
- 整体风格定位
- 目标受众
- 更新频率

### 4.2 workflow

以分类清单形式保存制作总流程：

- 前期策划
- 剧情改编
- 视觉制作
- 音频制作
- 视频合成

每类保存多条布尔状态项。

### 4.3 episodes

章节改编条目为数组，每条包含：

- 卷名
- 章节名
- 原文范围
- 本集时长目标
- 一句话梗概
- 核心看点数组
- 情绪曲线
- 备注

### 4.4 characters

角色条目为数组，每条包含：

- 角色名
- 身份
- 年龄外观
- 性格关键词
- 人物定位
- 外貌特征
- 服装风格
- 代表色
- 常用表情
- 常用动作
- 声音类型
- Prompt
- 准备项清单
- 图片数组

### 4.5 scenes

场景条目为数组，每条包含：

- 场景名称
- 用途
- 场景描述
- 时间
- 氛围
- 画风要求
- 颜色关键词
- Prompt
- 准备项清单
- 图片数组

### 4.6 props

道具和特效条目为数组，每条包含：

- 名称
- 归属角色
- 用途
- 外观描述
- 材质颜色
- 特效描述
- Prompt
- 图片数组

### 4.7 storyboards

分镜脚本为数组，每条对应一个镜头：

- 镜头号
- 时间
- 景别
- 画面内容
- 人物动作
- 台词或旁白
- 音效或 BGM
- 备注
- 参考图数组

### 4.8 prompts

保存三类 Prompt 模板：

- 人物 Prompt
- 场景 Prompt
- 镜头 Prompt

每类包含正向提示词和负向提示词或补充字段。

### 4.9 voiceLines

配音台词为数组，每条包含：

- 序号
- 角色
- 台词内容
- 情绪
- 时长
- 是否已录制

### 4.10 audio / editing / quality

这些模块使用“分组清单 + 辅助表格字段”的方式保存：

- audio：背景音乐、音效勾选项
- editing：画面处理、字幕处理、音频处理、导出设置
- quality：内容检查、视觉检查、音频检查、发布检查

### 4.11 folders

保存推荐项目文件夹结构，可编辑：

- 文件夹名
- 说明

### 4.12 demoChecklist

保存“最简开工版模板”的勾选项。

### 4.13 meta

记录页面元信息：

- 当前版本
- 最近保存时间
- 是否使用默认模板初始化

## 5. 交互设计

### 5.1 标签切换

- 点击标签切换当前模块。
- 小屏幕下标签支持横向滚动。

### 5.2 条目管理

对于章节、角色、场景、道具、分镜、台词这类数组数据：

- 支持新增一条
- 支持复制当前条目
- 支持删除当前条目
- 支持折叠和展开条目

### 5.3 表单编辑

- 输入框适合短文本字段。
- 文本域适合描述、Prompt、梗概等长文本。
- 长文本域自动适应高度。
- 复选框用于清单项与布尔状态。

### 5.4 图片上传与预览

图片交互必须支持：

- 选择本地图片文件
- 将图片以 Data URL 形式写入当前状态
- 在页面中即时预览缩略图
- 删除已上传图片

图片使用场景包括：

- 角色设定图
- 场景参考图
- 道具特效图
- 分镜参考图

### 5.5 本地自动保存

- 任意字段修改后触发节流保存。
- 数据保存在浏览器 `localStorage`。
- 页面初始化时优先读取本地数据。
- 若无本地数据，则载入内置模板。

### 5.6 导入导出

- 导出：将完整状态树下载为 JSON 文件。
- 导入：读取用户本地 JSON 文件并覆盖当前状态。
- 导入前给出确认提示，避免误覆盖。

### 5.7 打印

- 支持调用浏览器打印。
- 打印样式优先保留文字与关键结构，隐藏按钮和上传控件。

## 6. 默认模板策略

页面首次打开时应自动注入一份可编辑默认模板，而不是空白结构。

默认模板内容来源于用户提供的漫剧清单，包含：

- 基础信息空字段
- 制作流程复选项
- 至少 1 个章节条目
- 至少 1 个角色条目
- 至少 1 个场景条目
- 至少 1 个道具条目
- 至少 5 行分镜空白模板
- 配音台词示例表头数据
- 音乐、剪辑、质检、Demo 清单
- 推荐项目文件夹结构

## 7. 视觉风格

页面风格需要适合内容生产工作流，不做花哨营销页风格。

视觉方向：

- 浅色背景为主，便于长时间编辑
- 带一点古风创作感，但不过度装饰
- 信息层级清晰，适合高密度表单
- 卡片、标签、按钮需要清楚区分主次

样式要求：

- 桌面端优先，宽屏下保持良好利用率
- 移动端可用，内容允许纵向堆叠
- 图片预览网格清晰
- 表格区域支持横向滚动

## 8. 技术实现

采用单文件原生前端实现：

- HTML：页面结构
- CSS：样式与响应式布局
- JavaScript：状态管理、渲染、事件处理、保存与导入导出

不使用框架，不依赖外部 CDN。

建议实现方式：

- 使用一个全局 `state` 对象
- 使用模板函数渲染不同模块
- 使用事件委托处理按钮点击
- 使用通用表单绑定函数同步数据
- 使用统一的图片上传处理函数

## 9. 错误处理

页面至少需要处理以下情况：

- 导入文件格式错误
- 导入 JSON 结构缺失
- localStorage 写入失败
- 图片文件读取失败
- 用户误删除条目

处理原则：

- 优先使用浏览器原生提示或页面内轻提示
- 不做复杂弹窗系统
- 错误信息用中文说明

## 10. 验收标准

交付的 HTML 文件应满足以下标准：

- 双击可直接打开使用
- 无网络时可正常编辑
- 支持标签切换
- 支持动态新增和删除多类条目
- 支持图片上传与预览
- 支持本地自动保存
- 支持 JSON 导入导出
- 支持打印为纸质清单或 PDF
- 页面打开后即可看到默认模板数据

## 11. 风险与取舍

### 11.1 图片持久化体积

图片以 Data URL 存入浏览器本地时，存储体积较大，可能受浏览器本地存储额度限制。

本次取舍：

- 优先保证离线可用和易用性。
- 默认提示用户不要一次性导入过多超大图片。
- 若达到浏览器存储上限，仍允许导出 JSON 进行备份。

### 11.2 单文件复杂度

单文件 HTML 在功能增多后会变大，但对于当前“即开即用”的目标是合理取舍。

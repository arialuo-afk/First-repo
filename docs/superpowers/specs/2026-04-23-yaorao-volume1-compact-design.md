# 《妖娆召唤师》第一卷紧凑化工作台设计

## 目标

在现有单文件 `HTML` 工作台上完成两件事：

1. 改成更紧凑、更容易定位信息的编辑模式。
2. 依据《妖娆召唤师》第一卷《朱雀异世》内容，按漫剧制作清单要求完成结构化归类填充。

本轮不处理用户数据持久化策略的修复，只调整页面结构与默认示例信息。

## 设计决策

### 1. 页面紧凑化

- 删除“文件夹模板”分栏，减少导航噪音。
- 保留单标签页工作台结构，但压缩顶部、卡片、表单和网格留白。
- 让高密度内容模块优先呈现“标题 + 摘要 + 可展开正文”的阅读方式。
- 默认示例中的重复条目多数折叠，只保留每个大类的前 1-2 条展开，方便扫读。

### 2. 信息归类方式

第一卷不按 40 章逐章平铺，而按剧情弧线归并为较少的“章节改编单元”，避免页面再次变成长文档。

建议归类为 6 段：

1. 荒野认父与黑珠净化（001-003）
2. 荒野训练与出谷准备（004-007）
3. 离谷入世与出云学院登场（008-010）
4. 招生试炼与水下宫殿支线（011-018）
5. 丹系成长、幻器线索与属性暴露（019-029）
6. 魔云宗潜伏、龙觉、纳多多与炎凰通灵（030-040）

### 3. 各模块填充原则

- `projectInfo`：按第一卷整体项目填写，不再限定为前三章试播。
- `workflow`：标记已完成“原著梳理、剧情拆分、角色/场景/道具归类”等前置项。
- `episodes`：填入 6 个剧情弧线单元，每个单元提供范围、梗概、看点、情绪曲线。
- `characters`：补充第一卷核心角色与势力角色，不追求穷尽龙套。
- `scenes`：提炼第一卷高频主场景和高记忆点场景。
- `props`：归类第一卷中真正影响剧情推进或视觉识别的道具/特效。
- `storyboards`：保留示范级镜头组，覆盖第一卷关键视觉桥段。
- `prompts` / `voiceLines` / `audio`：升级为第一卷级别的参考模板，而非只服务前三章。

## 非目标

- 不新增后端或云端同步。
- 不修改导入导出格式。
- 不解决“示例 hash 覆盖用户保存数据”的逻辑问题。
- 不把第一卷 40 章全部展开成 40 张章节卡。

## 验收标准

- 页面中不再出现“文件夹模板”分栏。
- 页面样式明显更紧凑，重复卡片更容易扫读。
- 默认示例不再是“前 3 章”，而是完整的第一卷归类版。
- 第一卷核心信息在项目、章节、角色、场景、道具、分镜、Prompt、配音等模块均有对应内容。
